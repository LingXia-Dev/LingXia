# Quick Start

The fastest path from zero to a running LingXia app.

This guide is the human onramp — toolchains, CLI install, your first project. If you'd rather have your AI coding assistant walk you through the framework, jump to [§5 Working with an AI assistant](#5-working-with-an-ai-assistant) — LingXia ships a markdown skill that agent tools such as Claude Code, OpenAI Codex CLI, and Cursor can consume.

---

## 1. Prerequisites

Core tools (always needed):

- **Node.js** 18 or later

Platform toolchains — install only the ones you target:

| Target | Toolchain |
|---|---|
| Android | Android SDK + NDK (command-line tools), JDK |
| iOS / macOS | Xcode on a macOS host |
| Harmony | Harmony command-line tools SDK |
| Windows native host app | Rust + MSVC Build Tools / Visual Studio on Windows |

If you'll write Rust native code (custom host APIs, native-only host), also install a **Rust** toolchain via [rustup](https://rustup.rs).

### Android command-line tools (one-time)

Download: https://developer.android.com/studio#command-tools

```bash
export ANDROID_SDK_ROOT=$HOME/android-sdk
export ANDROID_NDK_ROOT=$ANDROID_SDK_ROOT/ndk/28.2.13676358

$ANDROID_SDK_ROOT/cmdline-tools/latest/bin/sdkmanager --install \
  "build-tools;34.0.0" \
  "platform-tools" \
  "platforms;android-33" \
  "ndk;28.2.13676358"
```

Add the exports to your shell profile so every session has them.

### Harmony command-line tools (one-time)

Download: https://developer.huawei.com/consumer/en/download/command-line-tools-for-hmos

```bash
export OHOS_NDK_HOME=$HOME/OpenHarmony/command-line-tools/sdk/default/openharmony
```

## 2. Install the LingXia CLI

**macOS / Linux** (or Git Bash, MSYS, Cygwin on Windows):

```bash
curl -fsSL https://raw.githubusercontent.com/LingXia-Dev/LingXia/main/install.sh | sh
lingxia version
```

**Windows PowerShell:**

```powershell
irm https://raw.githubusercontent.com/LingXia-Dev/LingXia/main/install.ps1 | iex
lingxia version
```

Both installers drop the binary in `~/.local/bin` (`%USERPROFILE%\.local\bin`
on Windows). The PowerShell installer adds that directory to your user `PATH`
automatically; open a new terminal afterward so the change is picked up.
`install.sh` does not edit your profile; if `~/.local/bin` isn't already on
`PATH` it prints the `export PATH=...` line to add to your shell profile.

---

## 3. Verify your environment

```bash
lingxia doctor
```

Scope checks to one platform:

```bash
lingxia doctor --platform android
```

Fix anything reported as **fail** before moving on.

---

## 4. Create your first project

LingXia has two project shapes. Pick one:

### Standalone lxapp (page-based mini-app)

```bash
lingxia new my-lxapp -t lxapp -y
cd my-lxapp
lingxia dev
```

`lingxia dev` builds the app and launches LingXia Runner on the local desktop
runner platform (macOS or Windows).

### Native host app (Android / iOS / macOS / Harmony / Windows)

```bash
lingxia new my-app -t native-app -p macos --package-id com.example.myapp -y
cd my-app
lingxia dev
```

`-p` accepts a comma-separated list: `-p android,ios,macos,harmony,windows` or
`-p all`.

## 5. Working with an AI assistant

The deep reference for LingXia lives as a portable markdown skill. It documents the decision tree (lxapp vs. host app), every CLI command, page authoring, native components, the `lx.*` Logic API, the Rust native surface, and the bridge mechanics — organized so an AI agent can route to the right section instead of reading everything at once.

The skill is **opt-in**, distributed as an npm package. `lingxia new` prints a hint at the end of scaffolding but does not install the skill itself — run the command yourself when you want it:

```bash
npx @lingxia/skill install              # writes to ./.claude/skills/lingxia/
npx @lingxia/skill install --user       # writes to ~/.claude/skills/lingxia/
npx @lingxia/skill install --agents-md  # also writes AGENTS.md for Codex-style tools
npx @lingxia/skill@<version> install    # pin a specific version
```

The content is plain markdown laid out in the [Anthropic Skills convention](https://docs.claude.com/en/docs/claude-code/skills), with a `SKILL.md` entrypoint and relative reference files.

- **Claude Code, Claude Agent SDK** — use `npx @lingxia/skill install`; Claude discovers `.claude/skills/lingxia/SKILL.md`.
- **OpenAI Codex CLI** — use `npx @lingxia/skill install --agents-md`; Codex reads `<project>/AGENTS.md`, and the installer writes a pointer from that file to `.claude/skills/lingxia/SKILL.md`.
- **Cursor, GitHub Copilot, and other markdown-reading agents** — either point the tool at `.claude/skills/lingxia/SKILL.md`, or keep the `AGENTS.md` pointer generated by `--agents-md`.

You can browse the same content in this repo at [`docs/skill/SKILL.md`](./skill/SKILL.md) without installing anything.
