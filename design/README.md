# design/

Shared visual design sources for LingXia. Files here are masters: platform
SDKs, native chrome, docs, and marketing surfaces render or copy their final
assets from these sources.

This directory is not an app asset bucket. Do not put example resources,
lxapp page images, screenshots, fixtures, generated bundles, or product-specific
assets here. Those belong with the app, example, package, or generated platform
output that owns them.

| Directory | Contents |
| --- | --- |
| `app-icon/` | Brand app icon masters and rendered previews. |
| `banner/` | README/banner source and render. |
| `icons/` | Shared UI icon masters for LingXia SDK and native host chrome across platforms. |
| `og/` | Website/social preview source. |

## Shared UI Icons

`icons/svg/` is the source of truth for common SDK UI icons, such as navigation,
browser overlay, media controls, camera controls, and capsule/menu actions.

Platform generation uses these masters:

- Apple: PDF resources generated by the SDK release script.
- Android: vector drawables generated by the SDK release script.
- Harmony: rawfile SVG resources generated by the SDK release script.
- Windows: PNGs generated by the CLI into each app's host assets; the Windows
  SDK consumes those runtime assets instead of carrying a private icon copy.

Keep these icons generic and platform-neutral. If an icon is specific to an
example lxapp, a demo page, a product screen, or app content, keep it outside
`design/icons/`.
